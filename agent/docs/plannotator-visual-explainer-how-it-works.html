Execution path

Wrapper skill reads upstream workflow, then applies local theme and component rules.
Mental model

Thin wrapper, not rewrite

Skill does not replace Nico's visual-explainer. It routes through it, then swaps in Plannotator's design tokens and extra components so final HTML feels native inside the review / annotate browser UI.

  • reads base visual-explainer skill
  • reads local theme override reference
  • reads local extended pattern reference
  • outputs standalone HTML
  • renders through plannotator annotate --render-html

03

Preview path

Yes, preview comes from Plannotator server URL

You do not open this file directly in the browser. You start an annotation session, and Plannotator serves a browser UI on its local review/annotate server. That server URL is what you open.

/plannotator annotate docs/plannotator-visual-explainer-how-it-works.html --render-html
3
Open returned URL
Plannotator prints clickable local/public URL depending on local/SSH/Coder environment.

Bundled runtime state

What is bundled today

This Pi runtime now bundles the plannotator-visual-explainer wrapper skill under src/resources/skills/. That means the wrapper itself ships with the runtime and is discovered automatically like other bundled skills.

It packages the Plannotator-specific entrypoint, theme override, extended patterns, and HTML delivery conventions. The skill's job is to produce Plannotator-native visual explainers rendered through plannotator annotate --render-html.
