# Ghostty Theme Notes

Ghostty loads named themes from user config first, then from bundled resources. The lookup order and paths are implemented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/theme.zig:8-89`: user themes live under `ghostty/themes` in the XDG config directory (`theme.zig:28-35`), and resource themes live under `<resources_dir>/themes` (`theme.zig:57-60`). `theme.zig:16-18` explicitly allows installs without a resources directory.

The `theme` config documentation in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:548-586` repeats the same search order, states that built-in themes install under `Ghostty.app/Contents/Resources/ghostty/themes` on macOS and `share/ghostty/themes` on Linux, and documents light/dark pairing syntax as `light:theme-name,dark:theme-name`. Window theme mode values are documented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:2111-2121`.

Theme files are Ghostty config fragments. The cached checkout includes config test theme fixtures at `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/testdata/theme_dark`, `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/testdata/theme_light`, and `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/testdata/theme_simple`.

Core color tokens are ordinary config keys. `selection-foreground` and `selection-background` are documented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:699-707`. `palette = N=COLOR`, `palette-generate`, and `palette-harmonious` are documented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:777-835`. `cursor-color` is documented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:837-850`. `cursor-text` is documented in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:895-901`.

Palette entries use `palette = N=COLOR`, where `N` is `0..255`, and colors are `#RRGGBB`, `RRGGBB`, or named X11 colors. Ghostty stores a full 256-color palette and tracks explicitly set indices, shown in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/config/Config.zig:5829-5875`.

libghostty-vt exposes runtime setters for terminal theme colors. `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/include/ghostty/vt/terminal.h:96-121` describes foreground, background, cursor, and 256-color palette defaults plus OSC overrides. `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/include/ghostty/vt/terminal.h:500-532` documents `GHOSTTY_TERMINAL_OPT_COLOR_FOREGROUND`, `GHOSTTY_TERMINAL_OPT_COLOR_BACKGROUND`, `GHOSTTY_TERMINAL_OPT_COLOR_CURSOR`, and `GHOSTTY_TERMINAL_OPT_COLOR_PALETTE`. The C API does not expose a cursor text setter, so Android applies `cursor-text` in the GLES renderer for the glyph under the block cursor.

Ghostty applies configured palette, foreground, background, and cursor color to terminal state in termio derived configuration. Relevant flow is in `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/termio/Termio.zig:140-181`.

Bundled themes are provided as a lazy build dependency. `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/build.zig.zon:118-121` points `iterm2_themes` at `https://deps.files.ghostty.org/ghostty-themes-release-20260427-153600-5e4d1de.tgz`, and `/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/src/build/GhosttyResources.zig:128-135` installs that dependency into `share/ghostty/themes` when `emit_themes` is enabled.

The Android PoC themes in `app/src/main/res/raw/` are copied from that bundled theme archive: `Flexoki Light` and `Flexoki Dark`. They preserve Ghostty config-fragment syntax with foreground, background, `cursor-color`, `cursor-text`, selection, and ANSI palette entries. Ghostty is MIT licensed (`/Users/shady/.cache/checkouts/github.com/ghostty-org/ghostty/LICENSE:1-18`). The bundled theme archive is sourced from Ghostty's build metadata above.

Terminal fonts are also exposed as Android resources. `app/build.gradle.kts` vendors downloaded font archives into `build/generated/res/vendorFonts/font`, registers that directory as a generated resource source set, and Kotlin loads them through `R.font.*`.
